Challenge Open Data
Visualisation des données Documentation

Ce projet s'incrit dans le cadre de la 3ème année ISI à l' Ensimag. Le cours de Challenge Open Data a pour objectif de développer une application Web de visualisation interactive de données. Cette page Web a été conçu en une semaine par Léa BARRAU, Manon BLANCO, Alexandre PILLEYRE, Pedro Henrique PONS FIORENTIN et Léa SOLO KWAN

Jeux de données

Nos données sont servies par l’API de World Bank. Nous utilisons notamment les données concernant la pauvreté, les inégalités et l’environnement. Une description de leur API est disponible ici.

Données libres

Nous avons vérifié que les données utilisées sont libres. En effet, dans les conditions d’utilisation nous pouvons lire : You may extract, download, and make copies of the data contained in the Datasets, and you may share that data with third parties according to these terms of use. Malheureusement, nous n’avons pas pu vérifier les conditions d’utilisation de l’API puisque le lien mentionné dans leur documentation est invalide. Mais l’API étant utilisée par un groupe précédent nous en avons déduit que son utilisation était autorisée dans le cadre de ce projet.

Indicateurs et données utilisés

Nous utilisons les données provenant de différents indicateurs. Tous les indicateurs nous viennent de World Bank et sont disponibles dans la section Data Themes de leur site. Nous nous sommes intéressé en particulier aux données traitant :

  • des émissions de CO₂

Nous avons spécifiquement choisi ces indicateurs, car ils ont tous un lien avec le développement durable (notamment l’environnement) et la responsabilité sociale.

Traitements opérés

Pour des questions de pérennité, nous avons choisi d'avoir les données disponibles directement sur notre site. Nous avons choisi le format CSV pour répondre aux contraintes de taille de notre site. En effet, celui-ci doit faire moins de 10Mo.

Récupération des données

Dans un premier temps, nous avons téléchargé toutes les données au format CSV depuis le site de World Bank. Leur site propose d'accéder facilement à leurs données dans des formats prédéterminés (API, XLS, CSV, DataBank).

Nettoyage et agrégation des données

Afin de répondre aux contraintes de taille, nous avons décidé d'agréger toutes les données dans un unique fichier CSV afin d'éviter le dédoublement d'information. De plus, nous avons aussi réduit le nombre de données en limitant les dates utilisées. Les données disponibles sur notre site datent de 2010 au plus tôt. Ce dernier choix, nous permet aussi de nettoyer nos données puisque qu'avant 2010, les informations ne sont pas toujours présentes pour tous les pays. Pour réaliser cette tâche, nous avons écrit un script Python utilisant la librairie Pandas.

Choix des technologies

Notre site utilise évidemment les langagues Web de base (HTML, CSS et Javascript). Mais concernant les représentations graphiques nous avons dû faire certains choix techniques.

Heatmap et Graphiques

Pour la carte et les graphiques notre choix s'est arrêté sur la librairie D3.js. Cette librairie nous permet de facilement manipuler des documents à partir des données.

Style

Pour simplifier les développements, nous avons choisi d'utiliser le framework Bulma pour la présentation de notre page Web.

Architecture

Notre avons voulu que notre application reste minimale. Pour cela, nous avons découpé notre projet selon les langagues de notre projet. Nous avons donc:

  • un fichier index.html qui est l'entrée de notre page Web
  • un dossier js qui contient tous les fichiers Javascript
